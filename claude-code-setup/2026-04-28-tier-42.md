# Tier 42: Auto-memory

**Дата**: 2026-04-28
**Статус**: ✅ Реализовано
**Цель**: Автоматическое сохранение контекстных фактов (пользователь, проект, фидбек) без явного tool call.

## Зачем

Без auto-memory знания теряются между сессиями — агент сохраняет только то, что явно попросили (см. gap analysis). Auto-memory добавляет два механизма:
1. **LLM-driven**: system prompt instructs the agent to call `memory_set` automatically for important facts
2. **Extraction-driven**: post-response scanning для URL, decisions, ключевых паттернов

## Механизмы

### 1. System prompt injection

При создании новой сессии в `session.system` загружается промпт с:
- Инструкцией auto-save facts через memory_set
- Текущим контекстом памяти (последние 20 записей) — чтобы агент знал, что уже сохранено

Формат ключей: `user/`, `project/`, `decision/`, `reference/`, `feedback/`.

### 2. Post-response extraction (`_auto_memory_extract`)

После каждого финального ответа (без tool calls) ассистента запускается экстракция:
- **URLs**: `https?://(domain)` → `reference/{domain}`
- **Decisions**: ключевые фразы `using|decided to|lets use|going with|opted for|chose` + термин → `decision/{term}`

Дубликаты не перезаписываются.

### 3. Memory context loading

При старте сессии в system prompt загружаются 20 последних записей из `_memory`. Это даёт агенту контекст предыдущих сессий.

## Что сделано

### 1. State
```python
_AUTO_MEMORY_ENABLED = True
```

### 2. 2 функции

- `_auto_memory_extract(text, session_id)` — сканирует ответ ассистента, извлекает URL и decisions, сохраняет в `_memory`
- `_build_auto_prompt()` — собирает system prompt с инструкциями + контекстом памяти

### 3. 2 hook points в handle_chat

- После создания сессии: `session.system = _build_auto_prompt()`
- После финального ответа: `_auto_memory_extract(content, session_id)`

### 4. Version bump
- Tier 41 → Tier 42 (99 tools, auto-memory не добавляет инструментов)

## Тестирование

```bash
# Chat with facts
curl -X POST http://host:8766/chat \
  -H 'Content-Type: application/json' \
  -d '{"message": "My name is Alex and I work at ExampleCorp. Our site is https://example.com"}'

# Verify auto-saved memory
curl -X POST http://host:8766/tools/memory-search \
  -H 'Content-Type: application/json' \
  -d '{"prefix": ""}'
# → 4 entries: user/name=Alex, user/company=ExampleCorp, user/website=https://example.com,
#   reference/example.com={url, source: auto}
```
