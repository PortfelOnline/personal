# Tier 40: Sub-agents

**Дата**: 2026-04-28
**Статус**: ✅ Реализовано
**Цель**: Аналог Agent tool из Claude Code — делегирование подзадач в фоновые потоки с вызовом DeepSeek API.

## Зачем

Главный гэп до Claude Code (см. gap analysis). Без sub-agents агент не может:
- Параллельно выполнять независимые задачи
- Использовать специализированные модели (flash для быстрых задач, pro для сложных)
- Изолировать контекст между разными типами задач

## Архитектура

```
subagent_spawn(params)
  ├── task: str — инструкция субагенту
  ├── system: str — опциональный system prompt
  └── model: str — опциональный выбор модели (deepseek-v4-flash / deepseek-v4-pro)
       │
       ▼
  tid = "sa" + timestamp[-8:]  # уникальный ID
  _subagents[tid] = {task_id, task, system, model, status: "running", result: "", ...}
       │
       ▼
  threading.Thread(target=_subagent_worker, daemon=True)
       │
       ▼
  _subagent_worker()
    ├── вызывает chat(task, system, model)
    ├── успех → status="completed", result=ответ
    └── ошибка → status="failed", result=str(error)
```

### Thread Safety

- `_subagents = {}` — общий dict
- `_subagents_lock = threading.Lock()` — threading.Lock для всех чтений/записей
- Все 4 инструмента и worker используют `with _subagents_lock:`
- Daemon-потоки — не блокируют завершение процесса

### 4 инструмента

| Инструмент | Параметры | Описание |
|-----------|-----------|----------|
| `subagent_spawn` | `task`, `system`?, `model`? | Запустить субагента в фоне |
| `subagent_list` | `status`? | Список субагентов (с фильтром) |
| `subagent_result` | `task_id` | Получить результат |
| `subagent_kill` | `task_id` | Убить запущенного субагента |

## API

```bash
# Spawn
curl -X POST http://host:8766/tools/subagent-spawn \
  -H 'Content-Type: application/json' \
  -d '{"task": "Analyze this log file", "system": "You are a log analyzer"}'
# → {"task_id": "sa73824875", "status": "running", "task": "Analyze this log file"}

# List
curl -X POST http://host:8766/tools/subagent-list \
  -H 'Content-Type: application/json' \
  -d '{"status": "completed"}'
# → {"total": 1, "summary": {"running": 0, "completed": 1, ...}, "subagents": [...]}

# Result
curl -X POST http://host:8766/tools/subagent-result \
  -H 'Content-Type: application/json' \
  -d '{"task_id": "sa73824875"}'
# → {"task_id": "...", "status": "completed", "result": "Found 3 anomalies...", ...}

# Kill
curl -X POST http://host:8766/tools/subagent-kill \
  -H 'Content-Type: application/json' \
  -d '{"task_id": "sa73825257"}'
# → {"task_id": "...", "status": "killed"}
```
