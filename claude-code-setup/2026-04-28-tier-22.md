# Tier 22: Web Tools — HTTP-инструменты для DeepSeek Agent

**Дата**: 2026-04-28
**Статус**: ✅ Реализовано
**Цель**: DeepSeek Agent может ходить в веб, как Claude Code.

## Зачем

Сейчас агент не может:
- Читать веб-страницы по URL
- Искать в Google/Яндексе
- Проверять API-эндпоинты

## Что сделать

### 1. Инструмент `web_fetch(url, selector)`
- HTTP GET с User-Agent, парсинг HTML → текст
- Опциональный CSS selector для целевого контента
- Таймаут 10-15 секунд

### 2. Инструмент `web_search(query)`
- DuckDuckGo без API-ключа (или платный SerpAPI/DataForSEO)
- Возврат топ-N результатов с заголовками и сниппетами

### 3. Безопасность
- Таймауты, ограничение размера ответа (50KB)
- Нет доступа к internal IP (127.0.0.1, 10.x, 172.x, 192.168.x)
- rate limit: не более 10 запросов в минуту

## Зависимости

- `safe_access.py` — `check_network_access()` уже есть (Tier 21)

## Что сделано

### 1. `tool_web_fetch(url, selector)`
- HTTP GET через `requests` с User-Agent `Mozilla/5.0 (compatible; DeepSeekAgent/1.0)`
- BeautifulSoup парсинг HTML → читаемый текст
- Авто-удаление `<script>`, `<style>`, `<nav>`, `<footer>`, `<header>`, `<aside>`
- Опциональный CSS selector для целевого контента
- Таймаут 15s, лимит ответа 50KB
- Обработка Timeout / ConnectionError / HTTPError с читаемыми сообщениями

### 2. `tool_web_search(query, max_results)`
- DuckDuckGo Lite (HTML-эндпоинт `https://lite.duckduckgo.com/lite/`) — работает из датацентров, в отличие от API
- BeautifulSoup парсинг таблицы результатов (table[2], 4 строки на результат)
- Извлечение реального URL из DDG-редиректа (`uddg` параметр)
- max_results: 1-10 (default 5)

### 3. Безопасность (`safe_access.py`)
- `check_web_url()`: только http/https, блокировка internal IP (127.0.0.1, 10.x, 172.x, 192.168.x)
- `check_web_rate_limit()`: скользящее окно 60s, не более 10 запросов/минуту
- Цепочка вызова: validate URL → check rate → execute

### 4. Регистрация
- `TOOL_DEFS`: JSON-schema для tool calling LLM
- `TOOL_DISPATCH`: `"web_fetch": tool_web_fetch, "web_search": tool_web_search`
- `TOOL_URL_MAP`: `"web-fetch": "web_fetch", "web-search": "web_search"`

## Проверка

```json
// web_search("python programming", 3) → 3 результата
// web_fetch("https://httpbin.org/get") → JSON ответ
// web_fetch("http://127.0.0.1:8766/health") → blocked (internal IP) ✅
// web_fetch("file:///etc/passwd") → blocked (non-http) ✅
// web_fetch("https://example.com", selector="h1") → "Example Domain" ✅
```

## Проблемы и решения

| Проблема | Решение |
|----------|---------|
| duckduckgo_search.DDGS → 0 results из датацентра | Переход на DDG Lite HTML endpoint |
| urllib.parse.urlparse → NameError | Замена на прямой import + вызов |
| SSH quoting с nested quotes | Python скрипт в /tmp → scp → python3 |
