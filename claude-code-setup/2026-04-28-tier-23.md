# Tier 23: MCP Client Env Vars + GitHub MCP

**Дата**: 2026-04-28
**Статус**: ✅ Реализовано
**Цель**: DeepSeek Agent поддерживает `${VAR}` подстановку в MCP config и может управлять GitHub репозиториями.

## Зачем

MCP серверы (GitHub, GitLab, PostgreSQL, Playwright) требуют API-ключи/токены. Без прокидывания `env` в subprocess они не работают.

## Что сделано

### 1. `mcp_client.py` — `connect_stdio()` env-прокси
- Подпись: `connect_stdio(command, args=None, env=None)`
- Внутри: копия `os.environ` + merge `env` dict → subprocess.Popen(env=proc_env)
- `init_mcp_servers()` передаёт `cfg.get("env")` в connect_stdio

### 2. `agent.py` — `${VAR}` resolution в `init_mcp()`
- После парсинга mcp.json: проход по всем `env.values()`
- Если значение вида `${VAR_NAME}` → `os.environ.get(VAR_NAME, "")`
- Иначе → pass as-is

### 3. Dockerfile — зависимости
- Добавлены `requests`, `beautifulsoup4` в pip install (требовались для Tier 22 web tools, но отсутствовали в образе)

### 4. GitHub MCP сервер
- Пакет: `@modelcontextprotocol/server-github` (v2025.4.8 — deprecated, но работает)
- mcp.json: `{"command": "mcp-server-github", "env": {"GITHUB_TOKEN": "${GITHUB_TOKEN}"}}`
- .env: `GITHUB_TOKEN=<token>` (из `~/.config/gh/hosts.yml`)
- docker-compose.yml: прокинут `GITHUB_TOKEN=${GITHUB_TOKEN}`

### 5. Результат
- 37 MCP tools от 2 серверов (filesystem + github)
- 49 total tools (12 native + 11 filesystem MCP + 26 github MCP)
- GitHub tools: create_issue, create_pull_request, create_branch, merge_pull_request, search_code, search_users, list_commits, list_issues, list_pull_requests, get_issue, get_pull_request, add_issue_comment, fork_repository, create_pull_request_review, update_pull_request_branch, get_pull_request_files/status/comments/reviews, search_repositories, update_issue, push_files, search_issues

## Проблемы и решения

| Проблема | Решение |
|----------|---------|
| `ModuleNotFoundError: requests` после rebuild | Добавлен в Dockerfile pip install |
| `server-github` deprecated на npm | Пока работает, миграция позже |
| docker-compose.yml pattern не matching | Прочитал точный файл через SSH → точный патч |
| gh auth token требует TTY | Чтение из `~/.config/gh/hosts.yml` напрямую |
