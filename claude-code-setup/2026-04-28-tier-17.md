## 1. Model Router (`route_model()`)

**Файл**: `~/.local/lib/deepseek_api.py`

Эвристики авто-роутинга (pro vs flash):
- JSON mode → pro (структурированный вывод)
- Temperature < 0.3 → pro (детерминизм)
- Системный промпт → pro (сложная инструкция)
- Промпт > 500 символов → pro (контекстная сложность)
- Ключевые слова: архитектур, рефакторинг, аудит, миграция, план, review, безопасность, оптимизаци, отлад, debug, scale, мониторинг, проект, дизайн, schema, интеграци, контракт → pro

Всё остальное → flash (экономия токенов, быстрее).

## 2. Health Aggregator (`/services`)

**Файл**: `/root/deepseek-agent/agent.py` (на сервере n)

Новый эндпоинт `GET /services` — агрегированный health всех 13 контейнеров:

```
deepseek-agent, kad, bot_dashboard, proxy_checker, aiwaiter,
yandex_bot, n8n, redis, browserless, qdrant, postgres,
mariadb-wordpress, buildx_buildkit
```

### Механика проверки

1. Docker socket API (`/containers/json?all=true`) — статус контейнеров
2. TCP SYN (`socket.connect_ex`) на exposed портах — доступность сервиса

Статусы: `ok` (контейнер up + порт отвечает), `degraded` (контейнер up но порт не слушает), `down` (контейнер не running).

### Docker-изменения

- `network_mode: host` — контейнер видит host loopback для TCP-проверок
- `volumes: /var/run/docker.sock` — доступ к Docker демону
- Docker socket API через `AF_UNIX` сокет (не требует docker CLI)

## 3. Daily Health Report

`nightly_cleanup()` теперь включает:
- Health check всех 13 сервисов
- Если есть down/degraded — отправляет alert
- AI summary включает статистику (диск + сервисы)

## 4. Команды

```bash
# Health check всех сервисов
ssh n "curl -s localhost:8766/services | python3 -m json.tool"

# Ручной запуск nightly cleanup
ssh n "curl -s -X POST localhost:8766/run/nightly-cleanup"

# Логи агента
ssh n "docker logs deepseek-agent --tail 30"

# Статус всех задач
ssh n "curl -s localhost:8766/tasks | python3 -m json.tool"
```
