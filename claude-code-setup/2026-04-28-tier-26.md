# Tier 26: Playwright MCP — Browser Automation

**Дата**: 2026-04-28
**Статус**: ✅ Реализовано
**Цель**: DeepSeek Agent может управлять браузером как Claude Code.

## Зачем

Без браузера агент не может:
- Проверять как выглядят веб-страницы (screenshot)
- Заполнять формы, кликать, навигировать в динамическом UI
- Тестировать веб-приложения
- Получать DOM-снэпшоты для анализа структуры страницы

## Что сделано

### 1. Dockerfile — npm + chromium
- `@playwright/mcp` добавлен в `npm install -g` (глобально в контейнере)
- `npx playwright install --with-deps chromium` — загрузка Chromium Headless Shell (112 MB) + системные зависимости
- Установлено поверх `python:3.10-slim`

### 2. mcp.json — Playwright MCP конфиг
```json
"playwright": {
  "command": "playwright-mcp",
  "args": [
    "--headless",
    "--no-sandbox",
    "--browser", "chromium",
    "--caps", "vision"
  ]
}
```
- `--headless` — нет дисплея в Docker
- `--no-sandbox` — требуется для Docker
- `--caps vision` — включена поддержка vision (screenshot → анализ)

### 3. Инструменты (28 browser tools)
- **Навигация**: `browser_navigate`, `browser_navigate_back`, `browser_reload`, `browser_tabs`
- **Клики/ввод**: `browser_click`, `browser_type`, `browser_fill_form`, `browser_select_option`, `browser_hover`, `browser_press_key`, `browser_drag`, `browser_drop`
- **Скриншоты/снэпшоты**: `browser_snapshot` (accessibility tree), `browser_take_screenshot`
- **Консоль/сеть**: `browser_console_messages`, `browser_network_requests`
- **Eval/код**: `browser_evaluate`, `browser_run_code`
- **Файлы**: `browser_file_upload`
- **Ожидание**: `browser_wait_for`
- **Диалоги**: `browser_handle_dialog`

### 4. Результат
- 5 MCP серверов (filesystem, github, postgres×2, playwright)
- 67 MCP tools (включая дубликаты query из 2 postgres серверов)
- 78 total tools (67 MCP + 11 native)
- +28 browser tools относительно Tier 25
