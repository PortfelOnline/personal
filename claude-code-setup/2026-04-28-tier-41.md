# Tier 41: Plan mode

**Дата**: 2026-04-28
**Статус**: ✅ Реализовано
**Цель**: Structured plan → user approval → implementation cycle (аналог EnterPlanMode из Claude Code).

## Зачем

Без plan mode агент начинает писать код сразу — нет структурированного планирования перед сложными изменениями (см. gap analysis). Plan mode добавляет цикл: explore → design → review → approve → implement.

## Архитектура

```
plan_create(params)          # → plan_id, draft
plan_section(params)         # → добавляет/обновляет секцию
plan_present(params)         # → status=review, возвращает форматированный план
plan_approve(params)         # → status=approved, → implement
plan_list(params)            # → список планов с фильтром по статусу
```

### 5 инструментов

| Инструмент | Параметры | Описание |
|-----------|-----------|----------|
| `plan_create` | `title`, `description?` | Создать новый план (draft) |
| `plan_section` | `plan_id`, `name`, `content` | Добавить/обновить секцию плана |
| `plan_present` | `plan_id` | Отметить как review, вернуть formatted |
| `plan_approve` | `plan_id` | Утвердить план → implement |
| `plan_list` | `status`? | Список планов (с фильтром) |

### Thread Safety

- `_plans = {}` — общий dict
- `_plans_lock = threading.Lock()` — все чтения/записи под Lock
- `_plan_mode_active = False` — флаг активного режима

### Plan lifecycle

```
draft → review → approved → implemented
  ↑         ↑         ↑
create   present   approve
```

## API

```bash
# Create
curl -X POST http://host:8766/tools/plan-create \
  -H 'Content-Type: application/json' \
  -d '{"title": "Add auth", "description": "Implement JWT auth"}'
# → {"plan_id": "pl832349", "title": "Add auth", "status": "draft", "sections": []}

# Section
curl -X POST http://host:8766/tools/plan-section \
  -d '{"plan_id": "pl832349", "name": "architecture", "content": "...design..."}'
# → {"plan_id": "pl832349", "section": "architecture", "added": True}

# Present
curl -X POST http://host:8766/tools/plan-present \
  -d '{"plan_id": "pl832349"}'
# → {"plan_id": "pl832349", "status": "review", "formatted": "# Plan: ..."}

# Approve
curl -X POST http://host:8766/tools/plan-approve \
  -d '{"plan_id": "pl832349"}'
# → {"plan_id": "pl832349", "title": "Add auth", "status": "approved"}

# List
curl -X POST http://host:8766/tools/plan-list \
  -d '{"status": "approved"}'
# → {"total": 1, "summary": {"approved": 1, ...}, "plans": [...]}
```

## Тестирование
