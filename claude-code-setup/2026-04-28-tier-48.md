# Tier 48: Async background tasks — BackgroundTaskManager + tools + HTTP endpoints

**Дата**: 2026-04-28
**Статус**: ✅ Реализовано
**Цель**: Добавить возможность запускать чат-задачи в фоне (async) — без блокировки HTTP response. Позволяет клиенту стартовать задачу, получить `task_id` и опрашивать статус позже.

## Зачем

Все chat-вызовы были синхронными — HTTP-соединение висело открытым до завершения ответа модели. Для длительных задач (анализ, генерация, multi-turn tool calls) это приводило к таймаутам на стороне клиента. Фоновые задачи решают эту проблему: клиент получает `task_id` сразу и забирает результат позже.

## Архитектура

### BackgroundTaskManager (`background_tasks.py`)

```
BackgroundTaskManager
  ├── _tasks: dict[str, dict]  — in-memory, thread-safe (Lock)
  ├── start(session_id, message, chat_func, system) → task_id
  ├── get(task_id) → entry | None
  ├── list(status, limit) → list[entry]
  └── cleanup_old(max_age_hours=24) → deleted_count
```

Жизненный цикл задачи:
```
start() → pending → _worker() → running → completed / failed
```

`_worker()` запускается в `threading.Thread(daemon=True)`:
1. Ставит статус `running`
2. Вызывает `chat_func(session_id, message)` — это `handle_chat()` из agent.py
3. Результат (JSON, до 5000 chars) сохраняется в `entry["result"]`
4. При ошибке: `entry["status"] = "failed"`, `entry["error"] = str(e)`

### API

**Tool-интерфейс** (через `/tools`):

| Tool | Описание |
|------|----------|
| `bg-task-start` | POST `{"message": "...", "session_id": "..."}` → `{"task_id": "bg_abc123", "status": "pending"}` |
| `bg-task-status` | POST `{"task_id": "bg_abc123"}` → полный entry с result/error |
| `bg-task-list` | POST `{"status": "completed", "limit": 20}` → список + summary по статусам |

**HTTP endpoint**:

| Method | Path | Описание |
|--------|------|----------|
| GET | `/bg-tasks` | Список всех задач (опционально `?status=completed`) |
| GET | `/bg-tasks/<id>` | Конкретная задача |

### Интеграция с agent.py

7 изменений:
1. `from background_tasks import BackgroundTaskManager` — импорт
2. `_bg_tasks = BackgroundTaskManager()` — глобальный инстанс
3. `tool_bg_task_start()`, `tool_bg_task_status()`, `tool_bg_task_list()` — 3 tool-функции
4. TOOL_URL_MAP: `bg-task-start` → `bg_task_start`, etc.
5. TOOL_DISPATCH: `bg_task_start` → `tool_bg_task_start`, etc.
6. GET `/bg-tasks` и GET `/bg-tasks/<id>` в do_GET
7. Version bump → `DeepSeek Agent v3 (Tier 48)`

## Тестирование

```bash
# Старт фоновой задачи
curl -s -X POST http://167.86.116.15:8766/tools/bg-task-start \
  -d '{"message": "привет", "session_id": "bg_test"}' | python3 -m json.tool
# → {"task_id": "bg_a1b2c3d4e5", "status": "pending"}

# Проверка статуса
curl -s -X POST http://167.86.116.15:8766/tools/bg-task-status \
  -d '{"task_id": "bg_a1b2c3d4e5"}' | python3 -m json.tool
# → {"task_id": "...", "status": "completed", "result": "{...}", ...}

# HTTP endpoint
curl -s http://167.86.116.15:8766/bg-tasks | python3 -m json.tool
curl -s http://167.86.116.15:8766/bg-tasks/bg_a1b2c3d4e5 | python3 -m json.tool
```
