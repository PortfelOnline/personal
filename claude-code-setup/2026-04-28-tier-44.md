# Tier 44: In-agent cron

**Дата**: 2026-04-28
**Статус**: ✅ Реализовано
**Цель**: Scheduled recurring tasks (cron) внутри агента — аналог CronCreate/CronList/CronDelete из Claude Code.

## Зачем

Без cron-инструментов агент не может сам планировать периодические задачи — всё зашито в код как статические scheduler.add_job(). In-agent cron позволяет агенту динамически создавать/удалять/просматривать расписания через function calling.

## Архитектура

Фактически cron_manager.py уже существовал (APScheduler + JSON persistence + CRUD). Не хватало только **3 инструментов** для вызова агентом через function calling:

```
cron_create(params)
  ├── expression: str — 5-польный cron ("0 9 * * *")
  ├── action: str — сообщение для handle_chat при срабатывании
  └── description: str?

cron_list(params)
  └── возвращает список задач с expression, next_run, description

cron_delete(params)
  └── task_id → удаляет из scheduler + JSON
```

### 3 инструмента (обёртки над cron_manager.py)

| Инструмент | Параметры | Что делает |
|-----------|-----------|------------|
| `cron_create` | `expression`, `action`, `description`? | Создать cron-задачу |
| `cron_list` | — | Список всех задач |
| `cron_delete` | `task_id` | Удалить задачу |

### Persistence

- JSON: `/app/data/user_cron.json`
- APScheduler: автоматически восстанавливается при старте (через `init_cron()`)

## Что сделано

### 1. Cron tools (функции-обёртки)

- `tool_cron_create(params)` — валидирует + вызывает `create_task()` из cron_manager
- `tool_cron_list(params)` — вызывает `list_tasks()` с next_run временем
- `tool_cron_delete(params)` — вызывает `delete_task()` из cron_manager

Все функции используют уже импортированные из `cron_manager`:
```python
from cron_manager import init_cron, create_task, list_tasks, delete_task
```

### 2. TOOL_DEFS — 3 definitions

```json
cron_create: {expression: string, action: string, description?: string}
cron_list:   {}
cron_delete: {task_id: string}
```

### 3. Регистрация (106 tools total)

- TOOL_DISPATCH — 3 entries
- TOOL_URL_MAP — 3 HTTP endpoints (`/tools/cron-*`)

### 4. Version bump

## Тестирование

```bash
# Create
curl -X POST http://host:8766/tools/cron-create
  -d '{"expression": "30 9 * * *", "action": "Run health check", "description": "Daily 9:30 UTC"}'
# → {"task_id": "task_0b6ff164d164", "status": "created", "next_run": "2026-04-29T09:30:00+00:00"}

# List
curl -X POST http://host:8766/tools/cron-list -d '{}'
# → {"total": 1, "tasks": [{...expression, next_run, description}]}

# Delete (error case)
curl -X POST http://host:8766/tools/cron-delete -d '{"task_id": "nonexistent"}'
# → {"error": "task not found: nonexistent"}

# Delete (success)
curl -X POST http://host:8766/tools/cron-delete -d '{"task_id": "task_0b6ff164d164"}'
# → {"task_id": "task_0b6ff164d164", "status": "deleted"}

# Verify empty
curl -X POST http://host:8766/tools/cron-list -d '{}'
# → {"total": 0, "tasks": []}
```
