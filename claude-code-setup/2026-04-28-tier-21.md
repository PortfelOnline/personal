# Tier 21: Context Compaction — Сжатие контекста

**Дата**: 2026-04-28
**Статус**: 🔜 В ПЛАНЕ
**Цель**: DeepSeek Agent не теряет контекст при длинных сессиях.

## Зачем

После N итераций tool calling история сообщений растёт → превышает контекстное окно → DeepSeek теряет нить диалога.

## Что сделать

### 1. Сжатие истории
- По достижении лимита (например, 80% context window) — суммаризировать ранние сообщения
- Заменить старые `user/assistant` сообщения одним summary-сообщением с system role

### 2. Truncation strategy
- Tail-truncate — сохранять последние K сообщений, удалять самые старые
- Lossy vs lossless: tool results можно дропать первыми (они тяжелее всего)

### 3. Формат compact-сообщения
  ```
  {"role": "system", "content": "Earlier conversation summary: ..."}
  {"role": "user", "content": "latest message"}
  ...
  ```
