# Tier 21: Context Compaction — Сжатие контекста

**Дата**: 2026-04-28
**Статус**: ✅ Реализовано
**Цель**: DeepSeek Agent не теряет контекст при длинных сессиях.

## Зачем

После N итераций tool calling история сообщений растёт → превышает контекстное окно → DeepSeek теряет нить диалога.

## Что сделано

### 1. Сжатие истории
- `Session.compact(chat_func)` — принимает функцию чата (DeepSeek flash)
- При достижении COMPACT_TRIGGER=15 сообщений запускается суммаризация
- Старые сообщения форматируются, передаются DeepSeek Flash с промптом:
  - "Keep: task goals, decisions, file paths. Discard: tool mechanics, errors, streaming tokens."
- Результат: `[Compact]: ...` сообщение заменяет все старые

### 2. Fallback
- При ошибке API — простая truncation (без суммаризации)

### 3. Интеграция
- `handle_chat()` — вызов `session.compact()` после добавления user message
- `handle_chat_stream()` — то же самое
- Используется `FAST_MODEL` (deepseek-v4-flash) — дёшево и быстро

### 4. Permissions UX (добавлено в этот же тир)
- **Dangerous command detection**: 12 regex-паттернов (rm -rf, dd, mkfs, fdisk, force push и т.д.)
- **Internal IP block**: localhost, 10.x, 172.x, 192.168.x, metadata
- **File size check**: предупреждение при чтении >10MB
- **Write size check**: предупреждение при записи >500KB
- Ответ в формате `{"permission_required": true, "warning": "..."}`
