# Tier 43: Monitor

**Дата**: 2026-04-28
**Статус**: ✅ Реализовано
**Цель**: Long-running process watcher (аналог Monitor tool из Claude Code) — фоновый запуск shell-команд с захватом вывода, проверкой статуса и возможностью остановки.

## Зачем

Без monitor агент не может отслеживать долгие операции — билды, деплои, обработку данных, хвосты логов. Все операции синхронные, блокирующие ответ. Monitor добавляет 4 инструмента для lifecycle: запустить → проверить → остановить → список.

## Архитектура

```
monitor_start(params)
  ├── command: string — shell-команда
  ├── timeout: int? — опциональный таймаут (сек)
       │
       ▼
  mid = "mo" + timestamp[-6:]   # уникальный ID
  _monitors[mid] = {command, status: "running", ...}
       │
       ▼
  threading.Thread(target=_monitor_worker, daemon=True)
       │
       ▼
  _monitor_worker(mid, command, timeout)
    ├── subprocess.Popen(shell=True)
    ├── proc.communicate(timeout=1) в цикле
    ├── проверка stop-флага каждую секунду
    ├── успех → status="completed"
    ├── ошибка → status="failed"
    └── stop signal → process.kill() → status="stopped"
```

### Thread Safety

- `_monitors = {}` — общий dict
- `_monitors_lock = threading.Lock()` — все чтения/записи под Lock
- Daemon-потоки — не блокируют завершение процесса

### 4 инструмента

| Инструмент | Параметры | Описание |
|-----------|-----------|----------|
| `monitor_start` | `command`, `timeout`? | Запустить shell-команду в фоне |
| `monitor_result` | `monitor_id` | Получить вывод, exit code, статус |
| `monitor_stop` | `monitor_id` | Остановить монитор (SIGKILL) |
| `monitor_list` | — | Список всех мониторов |

## Тестирование

```bash
# Start
curl -X POST http://host:8766/tools/monitor-start \
  -d '{"command": "echo hello && sleep 2 && echo done"}'
# → {"monitor_id": "mo838321", "status": "running", ...}

# Wait 3s...

# Result
curl -X POST http://host:8766/tools/monitor-result \
  -d '{"monitor_id": "mo838321"}'
# → {"status": "completed", "exit_code": 0, "stdout": "hello\ndone\n"}

# List
curl -X POST http://host:8766/tools/monitor-list -d '{}'
# → {"total": 1, "summary": {"completed": 1}, "monitors": [...]}

# Stop
curl -X POST http://host:8766/tools/monitor-start \
  -d '{"command": "sleep 60"}'
# → {"monitor_id": "mo838644", "status": "running"}

curl -X POST http://host:8766/tools/monitor-stop \
  -d '{"monitor_id": "mo838644"}'
# → {"status": "stopping", "message": "Stop signal sent"}
```
