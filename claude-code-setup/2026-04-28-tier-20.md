# Tier 20: Streaming — SSE для /chat

**Дата**: 2026-04-28
**Цель**: DeepSeek Agent стримит токены через SSE, как Claude Code.

## Зачем

Сейчас `/chat` ждёт полный ответ DeepSeek (иногда 10-30 сек). Claude Code streamит токены по мере генерации — пользователь видит процесс, а не белый экран.

## Что сделать

### 1. GET /chat/stream — SSE endpoint

Server-Sent Events на `GET /chat/stream?session_id=...&message=...`:
```
event: token
data: {"token": "..."}

event: done
data: {"response": "..."}

event: error
data: {"error": "..."}
```

### 2. Tool calling в stream режиме

При tool calling:
1. Стримить токены рассуждения DeepSeek
2. При `tool_calls` — отправить `event: tool_call`, выполнить, продолжить стрим
3. Финальный `event: done` с полным ответом

### 3. Формат SSE

```python
def send_sse_event(event, data):
    self.send_response(200)
    self.send_header("Content-Type", "text/event-stream")
    self.send_header("Cache-Control", "no-cache")
    self.send_header("Connection", "keep-alive")
    self.end_headers()
    # пишем events
    self.wfile.write(f"event: {event}\ndata: {json.dumps(data)}\n\n".encode())
    self.wfile.flush()
```
