# Tier 52: Claude Command Shell Fix

**Date:** 2026-04-29
**Context:** strategy-dashboard session, fixing developer environment

## Problem

The `claude` command (DeepSeek proxy wrapper) was **only defined in Claude Code's internal shell snapshot** at `~/.claude/shell-snapshots/`, not in `.zshrc`. Opening a new terminal gave `zsh: command not found: claude`.

## Root Cause

Claude Code saves the shell environment at session start into `~/.claude/shell-snapshots/snapshot-zsh-<timestamp>.sh`. Previous versions of this file contained the `claude()` function, but `.zshrc` did not. When `.zshrc` is sourced on terminal open, the function is simply not there.

## Fix

Added the complete `claude()` function to `~/.zshrc` (lines 118-137):

```zsh
# Claude Code ‚Äî –∑–∞–ø—É—Å–∫ —á–µ—Ä–µ–∑ DeepSeek –ø—Ä–æ–∫—Å–∏ (–∞–≤—Ç–æ-—Å—Ç–∞—Ä—Ç)
claude() {
  local key_file="$HOME/.config/deepseek-api-key"
  if ! lsof -ti:8099 &> /dev/null; then
    if [ ! -f "$key_file" ]; then
      echo "Error: $key_file not found" >&2
      return 1
    fi
    export DEEPSEEK_API_KEY="$(cat "$key_file")"
    python3 /Users/evgenijgrudev/personal/claude-code-setup/ds-proxy.py &
    sleep 2
    echo "‚Üí DeepSeek proxy started on :8099"
  fi
  export ANTHROPIC_BASE_URL=http://localhost:8099
  export ANTHROPIC_DEFAULT_SONNET_MODEL=deepseek-v4-flash
  export ANTHROPIC_DEFAULT_OPUS_MODEL=deepseek-v4-pro
  export ANTHROPIC_HAIKU_MODEL=deepseek-v4-flash
  unset ANTHROPIC_AUTH_TOKEN ANTHROPIC_API_KEY
  command claude "$@"
}
```

**What it does:**
1. Checks if DeepSeek proxy is already running on port 8099
2. If not, reads API key from `~/.config/deepseek-api-key`, starts `ds-proxy.py` in background
3. Sets `ANTHROPIC_BASE_URL=http://localhost:8099` to route through the proxy
4. Maps Claude models to DeepSeek models: sonnet/haiku ‚Üí `deepseek-v4-flash`, opus ‚Üí `deepseek-v4-pro`
5. Unsets any conflicting env vars (`ANTHROPIC_AUTH_TOKEN`, `ANTHROPIC_API_KEY`)
6. Runs actual Claude binary via `command claude "$@"`

## Breakage Mechanism (Anti-Pattern)

This is a class of breakage where **shell configuration lives in two places**: the runtime (Claude's snapshot) and the persistent startup file (`.zshrc`). Any shell function or alias defined via Claude Code that should survive across terminals must be exported to `.zshrc` (or equivalent). The snapshot mechanism preserves the *current* environment for the session, but does NOT automatically persist new functions to startup files.
