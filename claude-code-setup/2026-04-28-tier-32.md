# Tier 32: Scheduled Tasks API — Cron для DeepSeek Agent

**Дата**: 2026-04-28
**Статус**: ✅ Реализовано
**Цель**: API для создания/управления cron задачами пользователя, как Claude Code `CronCreate`/`CronDelete`.

## Зачем

Claude Code CLI поддерживает `CronCreate/CronDelete` — задачи по расписанию. Без этого агент не может:
- Запускать периодические проверки (health, диск, сервисы)
- Делать ежедневные отчёты в Telegram
- Запускать команды в определённое время
- Планировать обслуживание

## Что сделано

### 1. cron_manager.py — центральный модуль

Управление пользовательскими cron задачами:

```
cron_manager.py
├── init_cron(scheduler, on_trigger) → загрузка сохранённых задач в APScheduler
├── create_task(expr, action, session_id, desc) → создание новой задачи
├── list_tasks() → список всех задач (с next_run)
├── delete_task(tid) → удаление задачи
└── CRON_FILE = /app/data/user_cron.json
```

**Хранение**: JSON в `/app/data/user_cron.json`. Создаётся автоматически.

**Схема задачи:**
```json
{
  "task_<12-hex>": {
    "expression": "0 9 * * *",
    "action": "Получить статус сервера",
    "session_id": "cron_daily_report",
    "description": "Daily report",
    "created_at": "2026-04-28T11:40:00Z"
  }
}
```

**Лимиты**: макс 50 задач. Валидация cron expression (5 полей). Action обязателен.

### 2. API эндпоинты

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/cron` | Список задач (с next_run) |
| `POST` | `/cron` | Создать задачу |
| `DELETE` | `/cron/<id>` | Удалить задачу |

**POST /cron — тело запроса:**
```json
{
  "expression": "*/30 * * * *",
  "action": "Проверить свободное место на диске",
  "session_id": "cron_disk_check",
  "description": "Каждые 30 минут"
}
```

**Ответ:**
- 200: `{"id": "task_a7ac20b3e506"}`
- 400: `{"error": "invalid cron expression: ..."}`

### 3. Интеграция с agent.py

- `_cron_on_trigger(tid, cfg)` — callback при срабатывании задачи
- Запускает `handle_chat()` в отдельном треде с сохранёнными message + session_id
- Вызывается после `init_mcp()`, до логирования старта
- Автоматически загружает сохранённые задачи при старте агента

### 4. Dockerfile

`COPY cron_manager.py .` — baked into image.

## Тестирование

```bash
# Создать
curl -X POST http://localhost:8766/cron \
  -H "Content-Type: application/json" \
  -d '{"expression":"*/5 * * * *","action":"df -h","session_id":"cron_disk"}'
# → {"id": "task_..."}

# Список
curl http://localhost:8766/cron
# → [{"id":"task_...", "expression":"*/5 * * * *", "next_run":"...", ...}]

# Валидация
curl -X POST http://localhost:8766/cron \
  -H "Content-Type: application/json" \
  -d '{"expression":"bad","action":"test"}'
# → {"error": "invalid cron expression: ..."}

# Удалить
curl -X DELETE http://localhost:8766/cron/task_abc123
# → {"deleted": true}
```
