# Tier 34: Permission Gates — Confirmation Flow for Dangerous Operations

**Дата**: 2026-04-28
**Статус**: ✅ Реализовано
**Цель**: Добавить подтверждение пользователя перед опасными операциями (bash, write, edit).

## Зачем

Агент мог выполнить `rm -rf /`, перезаписать `/root/.ssh/authorized_keys`, отредактировать `/etc/` — и пользователь узнавал только по факту. Permission Gates добавляют двухфазный flow:

1. Агент пытается выполнить опасную операцию → `permission_required: true` + warning
2. Пользователь пишет "да"/"подтверждаю"/"yes" → агент повторяет вызов уже без блокировки
3. Любое другое сообщение → обычный чат, pending сбрасывается

## Что сделано

### 1. `safe_access.py` — новые паттерны и функции

**DANGEROUS_WRITE_PATTERNS** (6 паттернов):
| Паттерн | Что блокирует |
|---------|---------------|
| `/etc/` | Системные конфиги |
| `/root/.ssh/` | SSH ключи/доступ |
| `/root/.docker/` | Docker конфигурация |
| `id_rsa\|id_ed25519` | Приватные SSH ключи |
| `credentials.json` | Service account credentials |
| `/.env` | Secrets в .env |

**PERMISSION_POLICIES** — словарь всех правил для GET /permissions.

**`check_dangerous_write(path)`** — проверяет путь на опасные паттерны.

### 2. `agent.py` — механизм подтверждения

**PENDING_DANGEROUS dict** — per-session storage:
```python
session_id → {"type": "bash"|"write"|"edit", "key": md5(val)[:16], "params": {...}, "warning": str}
```

**Flow:**
```
User: "обнови конфиг nginx"
  → tool_bash('echo ... > /etc/nginx/nginx.conf')
  → check_dangerous_cmd → True, "Writing to /etc/..."
  → PENDING_DANGEROUS[sid] = {...}
  → {"permission_required": True, "warning": "...", "command": "..."}
  → LLM показывает пользователю: "Нужно разрешение: ..."
User: "да"
  → handle_chat: _is_confirmation("да") → True
  → injects auth system message: "[Permission granted...] Вызови тот же инструмент ещё раз."
  → LLM retries tool_bash с теми же params
  → check_dangerous_cmd → True → PENDING_DANGEROUS key совпадает → bypass
  → команда выполняется
```

**CONFIRM_KEYWORDS**: да, подтверждаю, разрешаю, выполняй, делай, давай, ок, ok, yes, confirm, go ahead, согласен, можно, выполнить.

**Метод проверки**: `t.strip().lower().rstrip(".!?") in CONFIRM_KEYWORDS`.

### 3. GET /permissions endpoint

Возвращает политики, лимиты и текущие pending операции (без params — только типы).
