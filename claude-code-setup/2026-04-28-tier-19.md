# Tier 19: MCP Client — DeepSeek Agent подключается к внешним серверам

**Дата**: 2026-04-28
**Статус**: ✅ Реализовано
**Цель**: DeepSeek Agent умеет подключаться к MCP-серверам — stdio JSON-RPC 2.0.

## Результат

DeepSeek Agent на `n:8766` подключается к MCP-серверам через протокол JSON-RPC 2.0 (stdio transport). Подключен **filesystem MCP сервер** — 14 инструментов (11 добавлены, 3 пропущены как дубликаты).

## 1. MCP Client (mcp_client.py)

**Файл**: `/root/deepseek-agent/mcp_client.py` (221 строка, stdlib-only)

`MCPClient` класс:
- `connect_stdio(command, args)` — spawn процесса, JSON-RPC по stdin/stdout
- `_initialize()` — MCP handshake (initialize → initialized notification)
- `list_tools()` — получение инструментов (с кешированием)
- `call_tool(name, arguments)` — вызов инструмента MCP
- `to_tool_def(mcp_tool)` — конвертация в OpenAI function calling формат
- `disconnect()` — корректное завершение процесса

Фоновые потоки:
- `_stderr_loop()` — логирование stderr сервера
- `_read_loop()` — чтение JSON-RPC ответов

## 2. MCP Server Discovery

`discover_servers()` читает `~/.claude/mcp.json`:
```json
{
  "mcpServers": {
    "filesystem": {
      "command": "mcp-server-filesystem",
      "args": ["/app", "/root", "/tmp"]
    }
  }
}
```

`init_mcp_servers()` подключается ко всем серверам, возвращает `(clients, extra_tool_defs)`.

## 3. Интеграция с agent.py

`init_mcp()` вызывается при старте:
- Читает `/root/.claude/mcp.json` (mount volume в контейнере)
- Подключается к каждому серверу
- Фильтрует дубликаты имён инструментов
- Регистрирует новые инструменты в `TOOL_DEFS` и `TOOL_DISPATCH`
- Создаёт `MCP_CLIENTS` для диспетчеризации

`tool_mcp(params)` — диспетчер:
- Префиксный формат: `mcp__<server>__<tool>` (HTTP API)
- Беcпрефиксный: поиск по всем клиентам (DeepSeek function calling)

## 4. Известные проблемы и фиксы

### 🔴 npm package names
**Симптом**: `npm ERR! 404 @anthropic/mcp-server-filesystem` — неправильные имена пакетов.
**Причина**: Пакеты под `@anthropic/` не существуют в npm registry.
**Фикс**: Использовать `@modelcontextprotocol/server-filesystem` и `@modelcontextprotocol/server-github`.

### 🔴 f-string syntax в Python 3.10
**Симптом**: `SyntaxError: f-string: invalid syntax` при запуске.
**Причина**: Python 3.10 не поддерживает кавычки (`" "` или `' '`) внутри f-string выражений, а также `\` escape-последовательности.
**Фикс**: Вынести сложные выражения в переменные, использовать конкатенацию строк вместо вложенных f-string.

### 🔴 Duplicate tool names
**Симптом**: `Tool names must be unique` — HTTP 400 от DeepSeek API.
**Причина**: MCP filesystem добавляет `read_file`, `write_file`, `edit_file` — те же имена, что у оригинальных HTTP-инструментов.
**Фикс**: Фильтрация дубликатов: `filtered = [ed for ed in extra_defs if ed["function"]["name"] not in existing_names]`.

### 🔴 MCP config not found inside container
**Симптом**: Лог "MCP config not found at ~/.claude/mcp.json".
**Причина**: `mcp.json` создан на хосте, но не смонтирован в контейнер.
**Фикс**: Volume mount в `docker-compose.yml`: `- /root/.claude/mcp.json:/root/.claude/mcp.json:ro`

## Архитектура

```
agent.py (init_mcp)
  └─→ mcp_client.py (discover_servers → init_mcp_servers)
       ├─→ MCPClient("filesystem") → connect_stdio("mcp-server-filesystem", [...])
       │    ├─→ _initialize()       # MCP handshake
       │    ├─→ list_tools()        # 14 tools
       │    └─→ TOOL_DISPATCH[name] = tool_mcp
       └─→ возвращает (clients, extra_defs)
            ├─→ MCP_CLIENTS = [...]
            └─→ TOOL_DEFS += filtered (11 новых)
```
