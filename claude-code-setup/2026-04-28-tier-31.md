# Tier 31: Hooks — Pre/Post Tool Processing для DeepSeek Agent

**Дата**: 2026-04-28
**Статус**: ✅ Реализовано
**Цель**: Система хуков — pre/post обработка вызовов инструментов, как в Claude Code.

## Зачем

Claude Code поддерживает hooks (`PreToolUse`, `PostToolUse`, `PreChat`, `PostChat`) для:
- Rate limiting — контроль частоты вызовов инструментов
- Логирования — запись времени выполнения и результатов
- Валидации — проверка аргументов перед вызовом
- Мониторинга — счётчики, алерты

Без хуков DeepSeek Agent не имел механизма перехвата/расширения поведения инструментов.

## Что сделано

### 1. hook_loader.py — центральный модуль

Аналог `plugin_loader.py`, но для событий:

```
hook_loader.py
└── discover_hooks() → сканирует hooks/*.py
└── load_hook() → importlib загружает модуль, валидирует HOOK_META + HOOKS
└── init_hooks() → discover + load + register, логирует сводку
└── run_hooks(event, **data) → итерирует все handler'ы события
└── list_hooks() → {event: [hook_names]} для API
```

**5 событий:**
| Событие | Вызывается | Pre может |
|---------|-----------|-----------|
| PreToolUse | Перед вызовом инструмента | Блокировать через `{"error": "..."}` |
| PostToolUse | После вызова инструмента | — |
| PreChat | Перед обработкой сообщения | Блокировать |
| PostChat | После ответа | — |
| PostInit | После инициализации агента | — |

**Pre-хуки** возвращают `None` (пропустить) или `{"error": "reason"}` (блокировать). Ошибки в хуках никогда не падают — `_safe_run_hooks()` ловит все exceptions.

### 2. Встроенные хуки

**log_tool_usage** (PostToolUse):
```python
HOOKS = {"PostToolUse": _on_post_tool}
# Формат: [Tool] HH:MM:SS tool_name(123ms) sid=xxx iter=2: result preview
```
Логирует каждый вызов инструмента: имя, время выполнения, session_id, итерацию, превью результата (первые 120 символов).

**rate_limit** (PreToolUse):
```python
HOOKS = {"PreToolUse": _on_pre_tool}
# Порог: 30 tool calls / 60s per session
```
Per-session rate limit. Скользящее окно 60 секунд, максимум 30 tool calls. Блокирует через error, если превышен.

### 3. Интеграция в agent.py

- `TOOL_DISPATCH` lookup обёрнут в `_safe_run_hooks("PreToolUse", ...)` — хук может заблокировать
- После выполнения — `_safe_run_hooks("PostToolUse", ...)` — логирование
- Timing: `duration_ms = int((time.time() - _start_tool) * 1000)` — передаётся в PostToolUse
- И `handle_chat`, и `handle_chat_stream` (SSE) используют одинаковый hook wrapper

### 4. API endpoint

`GET /hooks` → `{"hooks": {"PostToolUse": ["log_tool_usage"], "PreToolUse": ["rate_limit"]}}`

### 5. Dockerfile

`COPY hook_loader.py .` + `COPY hooks/ ./hooks/` — baked into image.

## Файлы

| Файл | Описание |
|------|----------|
| `hook_loader.py` | Центральный модуль: discovery, loading, execution |
| `hooks/log_tool_usage.py` | Хук: логирование tool calls |
| `hooks/rate_limit.py` | Хук: per-session rate limiting |
| `agent.py` | `_safe_run_hooks()`, интеграция в tool dispatch loop |
| `Dockerfile` | COPY hook_loader.py + hooks/ |
| `docker-compose.yml` | bind mounts (для разработки) |

## Результат

- **2 хука** загружены: log_tool_usage (PostToolUse), rate_limit (PreToolUse)
- **5 событий** в архитектуре: PreToolUse, PostToolUse, PreChat, PostChat, PostInit
- **Safe by design**: ошибки хуков не валят агент, Pre-хуки могут блокировать инструменты
- **Baked into image**: хуки работают даже без bind-монтирования
