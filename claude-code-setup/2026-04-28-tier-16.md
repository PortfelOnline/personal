# Tier 16 — DeepSeek Agent в Docker (24/7 облачное исполнение)

**Дата**: 2026-04-28
**Цель**: заменить Cloud Code 24/7 execution на Docker-контейнер с DeepSeek API на сервере `n`

## Архитектура

```
Сервер n (167.86.116.15)
└── /root/deepseek-agent/
    ├── Dockerfile              # python:3.10-slim + apscheduler
    ├── agent.py                # BackgroundScheduler + HTTP (:8766)
    ├── deepseek_api.py         # Копия библиотеки (stdlib-only)
    ├── docker-compose.yml      # container_name: deepseek-agent
    └── .env                    # DEEPSEEK_API_KEY (chmod 600)

Контейнер deepseek-agent (port 8766)
├── GET  /health    — status + uptime + tasks
├── GET  /tasks     — список задач с next_run
├── POST /run/<name> — ручной запуск задачи
├── nightly-cleanup  — 22:27 UTC ежедневно (3:57 IST)
└── weekly-refs-update — Пн 3:30 UTC (9:00 IST)
```

## Что решает

| Проблема Cloud Code | Решение Tier 16 |
|---------------------|-----------------|
| Требует ноутбук включённым | Контейнер 24/7 на VPS |
| Только через веб-интерфейс | HTTP API + docker exec |
| Cloudflare блокирует прямые запросы | Локальный API на сервере |
| Нельзя добавить свои инструменты | Полный доступ к хосту (при mount) |

## Технические детали

- **Python 3.10-slim** — 1 pip-зависимость (apscheduler)
- **APScheduler** — cron-подобный планировщик в процессе
- **stdlib http.server** — 3 эндпоинта, без FastAPI/Flask
- **HEALTHCHECK** — Docker native, каждые 60s
- **restart: unless-stopped** — авто-восстановление после ребута

## Команды

```bash
# Статус
curl localhost:8766/health

# Список задач
curl localhost:8766/tasks

# Ручной запуск
curl -X POST localhost:8766/run/nightly-cleanup

# Логи
docker logs deepseek-agent

# Перезапуск
cd /root/deepseek-agent && docker compose restart
```
