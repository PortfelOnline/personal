**Дата**: 2026-04-28
**Статус**: ✅ Реализовано
**Цель**: дать DeepSeek Agent'у на сервере n инструменты уровня Claude Code — файловые операции, git, bash, интерактивный /chat с tool calling.

DeepSeek Agent на `n:8766` получил **10 инструментов** + **multi-turn /chat** с полным циклом tool calling. Всё stdlib-only, без внешних зависимостей (кроме APScheduler).

## 4. Interactive Session (POST /chat)

Multi-turn диалог с DeepSeek V4 и автоматическим tool calling.

```json
POST /chat
{
  "session_id": "abc-123",  // опционально, создаётся если не указан
  "message": "прочитай файл config.yml и скажи что там",
  "temperature": 0.3         // опционально, по умолчанию 0.3
}
```

**Ответ**: `{"session_id": "...", "response": "...", "error": true|false}`

**Механика**:
- До 10 итераций tool calling loop
- На каждой итерации: DeepSeek → tool_calls → выполнение → результат → DeepSeek
- Сообщения ассистента сохраняются в сессии включая `tool_calls` и `reasoning_content`

**Сессионный API:**

| Endpoint | Метод | Назначение |
|----------|-------|-----------|
| `/chat` | POST | Отправить сообщение, получить ответ |
| `/sessions` | GET | Список активных сессий |
| `/sessions/{id}` | DELETE | Очистить историю сессии |

## 5. Session Management

- **SessionManager**: create/get/delete/list
- **Session**: история сообщений, `api_msgs()` в формате OpenAI API
- **MAX_MSGS**: 20 сообщений (с обрезкой старых при превышении)

## 6. Permission System (safe_access.py)

- **ALLOWED_PATHS**: `/app/`, `/root/`, `/tmp/`, `/var/log/`
- **BLACKLISTED_CMDS**: `rm -rf /`, `mkfs`, `dd`, fork bomb, `chmod -R 000 /`
- **BLOCKED_BASENAMES**: `.env`, `id_rsa`, `*.pem`, `credentials.json`, `*.key`
- **BLACKLISTED_PATH_PREFIXES**: `/proc/`, `/sys/`, `/dev/`, `/var/run/docker.sock`
- **safe_join()**: защита от path traversal

## 7. Tool Calling (OpenAI function calling format)

10 tool definitions в TOOL_DEFS для DeepSeek V4. Формат — стандартный OpenAI `tools` parameter.

```python
TOOL_DISPATCH = {
    "read_file": tool_read_file,
    "write_file": tool_write_file,
    "edit_file": tool_edit_file,
    "ls_dir": tool_ls_dir,
    "grep_files": tool_grep_files,
    "glob_files": tool_glob_files,
    "bash": tool_bash,
    "git_status": tool_git_status,
    "git_diff": tool_git_diff,
    "git_log": tool_git_log,
}
```

## 8. Известные проблемы и фиксы

### 🔴 reasoning_content (DeepSeek V4 thinking mode)

**Симптом**: HTTP 400 на втором вызове API после tool_calls.

**Ошибка**: `The reasoning_content in the thinking mode must be passed back to the API.`

**Причина**: DeepSeek V4 возвращает `reasoning_content` в ответе с tool_calls. При повторной отправке сообщения ассистента API требует включать `reasoning_content`.

**Фикс**:
1. `_api_call_with_tools()` — возвращает `reasoning_content` из ответа
2. `Session.add()` — принимает и сохраняет `reasoning_content`
3. `handle_chat()` — передаёт `reasoning_content` при сохранении сообщения с tool_calls

**Файлы**: `agent.py:447-449` (возврат), `session_manager.py:18-25` (хранение), `agent.py:489` (передача)

### 🔴 URL-to-dispatch mismatch

**Симптом**: `/tools/read` → `unknown tool: read`

**Причина**: URL-путь не совпадал с именем функции в TOOL_DISPATCH (`read` → `read_file`).

**Фикс**: Добавлен `TOOL_URL_MAP` — маппинг URL → dispatch name.

### 🔴 docker compose up не пересобирает образ

**Симптом**: правки agent.py не применялись после `docker compose up -d --force-recreate`.

**Фикс**: Использовать `docker compose up -d --build` или явный `docker build`.

## Проверено

- `/chat` read file → DeepSeek вызывает read_file → возвращает summary
- `/chat` write + read → 2 tool calls (write_file, read_file) → summary
- `/chat` bash (whoami, pwd) → bash tool → ответ
- `/chat` multi-step (ls → read → summarize) → 2+ итерации tool loop
- `/tools/read`, `/tools/write`, `/tools/ls`, `/tools/bash` — прямые HTTP
- `/health`, `/services`, `/sessions` — системные эндпоинты
- Session persistence между вызовами (same session_id)
- 400+ tool calls без ошибок reasoning_content
