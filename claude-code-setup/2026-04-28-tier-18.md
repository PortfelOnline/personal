# Tier 18: DeepSeek Agent → Claude Code Parity (MCP + Tools + REPL)

**Дата**: 2026-04-28
**Цель**: дать DeepSeek Agent'у на сервере n инструменты уровня Claude Code — MCP клиент, файловые операции, git, интерактивный режим.

## Текущий gap

Сейчас DeepSeek Agent — это HTTP API + шедулер. Он НЕ умеет:
- Читать/писать файлы (только `subprocess.run` + bash)
- Подключаться к MCP-серверам (GitHub, БД, FS)
- Работать интерактивно (multi-turn)
- Сжимать контекст при длинных сессиях

Claude Code имеет все эти возможности нативно.

## 1. MCP Client (главное)

**Что**: DeepSeek Agent подключается к MCP-серверам как клиент.

**MCP серверы для подключения:**
- `@anthropic/mcp-server-filesystem` — read/write/edit файлов
- `@anthropic/mcp-server-github` — PR, issues, commits (GitHub API v4)
- `@anthropic/mcp-server-postgres` — запросы к БД
- `@anthropic/mcp-server-memory` — краткосрочная память
- `@anthropic/mcp-server-puppeteer` — браузер (замена Playwright)

**Технически**: MCP — JSON-RPC 2.0 поверх stdio или SSE.
Достаточно одной зависимости `mcp` (Python SDK) или ручная реализация JSON-RPC.

```
agent.py
  └── MCPClient
        ├── connect("filesystem", ["/allowed/path"])
        ├── call("read_file", path="/etc/config.yml")
        └── call("search_files", pattern="*.ts", path="/project")
```

**Зависимости**: `pip install mcp` или stdlib-only JSON-RPC.

## 2. Filesystem Tools (структурированный доступ)

**Эндпоинты API** (добавить в HTTP Handler):

| Endpoint | Метод | Назначение |
|----------|-------|-----------|
| `/tools/read` | POST | Чтение файла |
| `/tools/write` | POST | Запись файла |
| `/tools/edit` | POST | Замена строки в файле |
| `/tools/ls` | POST | Список директории |
| `/tools/grep` | POST | Поиск по файлам |
| `/tools/glob` | POST | Поиск по паттерну |

Без MCP — реализовать как прямые функции в agent.py.
С MCP — прокидывать вызовы в MCP-сервер filesystem.

**Safe mode**: список разрешённых директорий (whitelist), запрет на `/etc/passwd`, `.env`, ключи.

## 3. Git Tools

| Endpoint | Метод | Назначение |
|----------|-------|-----------|
| `/tools/git-status` | GET | Статус репозитория |
| `/tools/git-diff` | GET | Diff изменений |
| `/tools/git-commit` | POST | Создать коммит |
| `/tools/git-push` | POST | Запушить |
| `/tools/git-log` | GET | История коммитов |

Через MCP GitHub сервер или прямые bash-вызовы.

## 4. Interactive Session (REPL endpoint)

**Что**: `POST /chat` — multi-turn диалог с сохранением истории.

```json
POST /chat
{
  "session_id": "abc-123",
  "message": "прочитай файл config.yml и скажи что там",
  "tools": ["mcp", "filesystem", "git"]
}
```

**Сохранение сессии**: in-memory dict (сессия → история сообщений).
**Ограничение**: max 20 сообщений, затем compaction.

**Сессионный API:**

| Endpoint | Метод | Назначение |
|----------|-------|-----------|
| `/chat` | POST | Отправить сообщение, получить ответ |
| `/sessions` | GET | Список активных сессий |
| `/sessions/{id}` | DELETE | Очистить историю сессии |

## 5. Context Compaction

При превышении лимита истории (20 сообщений / ~8k токенов):
1. Отправить историю в DeepSeek flash с промптом "сожми этот диалог в краткое резюме"
2. Заменить историю на резюме
3. Продолжить сессию

## 6. Permission System (Safety)

**Sandbox для инструментов:**
- Filesystem: только `/root/`, `/app/`, `/home/` — запрет `/etc/`, `/proc/`, `/sys/`
- Git: только репозитории в разрешённых директориях
- Bash: чёрный список команд (`rm -rf /`, `dd`, `mkfs`)
- По умолчанию: ALLOW. При高危операциях — ответ API с `"requires_confirm": true`
