# Tier 33: Multi-Model Routing — авто-выбор flash/pro для DeepSeek Agent

**Дата**: 2026-04-28
**Статус**: ✅ Реализовано
**Цель**: Авто-выбор модели DeepSeek (flash vs pro) в зависимости от сложности задачи, как в Claude Code.

## Зачем

Claude Code CLI сам выбирает модель (haiku/sonnet/opus) под задачу. DeepSeek Agent использовал только `DEFAULT_MODEL` (pro) или `FAST_MODEL` (flash) хардкодом. Без роутинга:
- Простые запросы (ls, grep, статус) шли на pro — переплата токенов
- Сложные задачи (рефакторинг, аудит) могли уйти на flash — терялось качество

## Что сделано

### 1. `route_model()` в серверный `deepseek_api.py`

Функция уже была в локальной копии (`~/.local/lib/deepseek_api.py`, Tier 17), но **отсутствовала на сервере**. Серверный `deepseek_api.py` — другая копия, без роутинга.

**Эвристики выбора pro:**
| Условие | Причина |
|---------|---------|
| `json_mode=True` | Структурированный вывод |
| `temperature < 0.3` | Детерминизм |
| Есть system prompt | Сложная инструкция |
| Промпт > 500 символов | Контекстная сложность |
| Ключевые слова (архитектур, рефакторинг, аудит, безопасност, etc.) | Требует глубокого анализа |

Всё остальное → flash.

### 2. Изменения в API

```python
# Было: всегда pro
def chat(prompt, system="", model=DEFAULT_MODEL, **kwargs):

# Стало: авто-роутинг
def chat(prompt, system="", model=None, **kwargs):
    if model is None:
        model = route_model(prompt=prompt, system=system, **kwargs)
```

То же для `stream_chat()`. Явный `model="deepseek-v4-pro"` всё ещё работает — override.

### 3. Интеграция в agent.py

`agent.py` импортирует `route_model`:
```python
from deepseek_api import chat, stream_chat, FAST_MODEL, DEFAULT_MODEL, route_model
```

Внутренние вызовы с `model=FAST_MODEL` (routing, planning) не менялись — там flash корректен.

### 4. Docker

Пересобрано через `docker compose up -d --build`. Без изменений в Dockerfile (только Python-код).
