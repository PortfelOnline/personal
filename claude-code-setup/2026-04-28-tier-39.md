# Tier 39: Notifications

**Дата**: 2026-04-28
**Статус**: ✅ Реализовано
**Цель**: Push-уведомления — сообщать пользователю о завершении долгих задач, ошибках, статусе.

## Зачем

Агент работает на сервере n (Германия), пользователь в Индии. Без нотификаций о завершении долгих операций (git push, батчи, deploy) приходится polling'ить API. Telegram-нотификации решают это.

## Что сделано

### 1. Инструмент `notify`

```python
tool_notify(params):
    message = params.get("message")
    level = params.get("level", "info")  # info|success|warning|error
    log(f"[NOTIFY:{level}] {message})      # всегда в лог
    _notify_telegram(message, level)       # если настроен Telegram
```

- Всегда пишет в лог Docker: `[NOTIFY:success] Deploy complete`
- Telegram-бэкенд: POST `https://api.telegram.org/bot{token}/sendMessage`
- Иконки: ℹ️ info, ✅ success, ⚠️ warning, 🚨 error
- Timeout: 10s на запрос к Telegram API
- Graceful fail: если Telegram не настроен, возвращает `telegram: false`

### 2. Конфигурация

**docker-compose.yml** (необходимо добавить):

```yaml
environment:
  - NOTIFY_TELEGRAM_BOT_TOKEN=${NOTIFY_TELEGRAM_BOT_TOKEN}
  - NOTIFY_TELEGRAM_CHAT_ID=${NOTIFY_TELEGRAM_CHAT_ID}
```

Или в `.env` рядом с `docker-compose.yml`:

```
NOTIFY_TELEGRAM_BOT_TOKEN=123456:ABC-DEF1234...
NOTIFY_TELEGRAM_CHAT_ID=123456789
```

### 3. API

```bash
# Простое уведомление
curl -X POST http://host:8766/tools/notify \
  -d '{"message": "Deploy завершён", "level": "success"}'
# → {"sent": true, "telegram": false, ...}

# Ошибка
curl -X POST http://host:8766/tools/notify \
  -d '{"message": "Бот упал: OOM", "level": "error"}'
```
