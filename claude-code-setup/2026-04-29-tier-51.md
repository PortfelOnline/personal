# Tier 51: API Key Auth + Multi-tenancy (opt-in, zero-risk)

**Дата**: 2026-04-29
**Статус**: ✅ Развёрнуто на сервере n (8766), container restarted, health OK
**Цель**: API key authentication with admin key seeding, key management tools, multi-tenant session isolation.

## Фичи

| # | Фича | Trigger | Default | Суть |
|---|------|---------|---------|------|
| 1 | **API Key Auth** | `AGENT_AUTH_ENABLED=1` | Выключена | Bearer token validation на `do_POST` и `do_DELETE` |
| 2 | **Admin key seed** | Первый запуск | Авто | `_seed_admin_key()` создаёт admin-ключ если keys пуст |
| 3 | **Key management** | Tools API | Всегда доступны | `create-api-key`, `revoke-api-key`, `list-api-keys` |
| 4 | **Session prefix** | При авторизации | Случайный суффикс | `_<random>` prefix для изоляции сессий по ключам |

## Реализация

- **Патч-скрипт**: `patches/tier51_auth.py` — 5 секций (константы+helpers, do_POST, do_DELETE, tool functions, dispatch entries)
- **Auth block** вставлен перед `scheduler = BackgroundScheduler(timezone="UTC")`
- **Ключи**: формат `ds_key_` + 32 hex chars (`secrets.token_hex(16)`)
- **Хранилище**: `/app/api_keys.json` (JSON, chmod 600)
- **Bind mount**: `./api_keys.json:/app/api_keys.json` в docker-compose.yml

## Исправления в процессе

1. `NameError: name 'secrets' is not defined` — добавлен `import secrets` в auth block
2. `IsADirectoryError: /app/api_keys.json` — Docker bind mount создал директорию вместо файла. Фикс: создан пустой `api_keys.json` на хосте перед стартом
3. `FileNotFoundError: /root/deepseek-agent/api_keys.json` — путь внутри контейнера должен быть `/app/api_keys.json`, не `/root/...`
