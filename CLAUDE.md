# 💰 TOKEN ECONOMY — HARD RULES
Эти правила обязательны. Нарушение = сжигание лимитов.

## 🚀 EARLY ANSWER MODE (приоритет #1)
**Pre-check logic (обязателен):**
Прежде чем использовать ЛЮБОЙ инструмент — остановись и оцени:
1. Могу ли я ответить прямо сейчас? (даже если не на 100% уверен)
2. Если ДА → отвечай. Инструменты = ЗАПРЕЩЕНЫ.
3. Если НЕТ → только один инструмент, не цепочку.
4. После получения ответа от инструмента — СРАЗУ отвечай, не делай второй вызов.

**Правило "короткий запрос":** если запрос пользователя < 200 символов → отвечай без инструментов.
**Правило "очевидное":** если задача тривиальная (опечатка, переименование, цвет, форматирование) → 0 шагов, сразу ответ.

## 📏 Лимиты (HARD STOP — хук exit 1)
- MAX_STEPS = adaptive (2-5, зависит от сложности запроса)
- MAX_FILES_PER_STEP = 3
- MAX_CALLS_PER_STEP = 2 (один и тот же tool 2+ раза подряд → STOP)
- MAX_TOTAL_CALLS = 5 (всего за сессию → STOP)
- MAX_TOKENS_PER_RESPONSE = ~2000
- Adaptive: короткие запросы → 2 шага, баги → 4, архитектура → 5

## 📖 Правила чтения
- НЕ читать файл целиком — только offset/limit нужной части
- Если файл >50 строк — offset обязателен
- НЕ перечитывать тот же файл дважды (exit 1)
- Анти-Partial-Read: прочитал кусок файла → не читай другой кусок
- graphify_query: budget ≤ 500 (simple=300, code=800, default=500)
- graphify: EMBEDDING_ONLY, DISABLE_FULL_TEXT

## 🛑 Tool chaining (запрещено)
НЕЛЬЗЯ делать цепочки инструментов в одном шаге:
- ❌ graphify → Read → curl → playwright
- ❌ curl site1 → curl site2 → curl site3
- ❌ Read file1 → Read file2 → Read file3
✅ Максимум: 1 инструмент на шаг, ответить, потом ещё 1 если нужно

## 🔄 Response cache
- Если тот же tool с тем же input вызван повторно — хук заблокирует (exit 1)
- Не пытайся "обойти" кэш меняя параметры — это loop detection
- Если ответ уже был дан на аналогичный запрос — используй его, не переспрашивай

## ⚡ Правила ответа
- Один ответ = одно дело. Без цепочек.
- НЕ retry loops. Ошибка → сообщи → STOP
- НЕ проверок, уточнений, исследований
- Если не уверен — ответь. Ошибка дешевле токенов на проверку.

## 🔄 Context fingerprinting
- Не пересылай одинаковые куски контекста
- Если файл уже в контексте — используй, не читай повторно
- Не дублируй информацию в ответе
- Ответ должен быть короче запроса (если这不是 ответ на сложный вопрос)

## 🌐 Tool discipline
- curl: 1 запрос = 1 цель. Без цепочек.
- MCP: 1 вызов = 1 ответ. Не вызывать то же самое повторно.
- Если инструмент вернул ошибку — не ретраить, сообщи и STOP

## 💵 COST MODEL (оценивай цену каждого шага)
Примерная стоимость действий в токенах:
- Read файла (100 строк): ~500-1000 токенов
- Bash(curl) / WebFetch: ~1000-3000 токенов (результат в контекст)
- WebSearch: ~2000-5000 токенов
- mcp__graphify__query: ~500-1500 токенов
- playwright/browser: ~3000-10000 токенов (скриншоты, HTML, консоль)

**Перед каждым действием: оцени, стоит ли результат этих токенов.**
Если ответ можно дать без инструмента — цена = 0. Выбирай 0.

## 📉 CONTEXT THRESHOLD — STOP при пустом результате
Если инструмент вернул мало данных (< 50 токенов полезной информации):
- **Первый раз:** дай 1 шанс на retry (возможно API вернул ошибку)
- **Второй раз подряд:** STOP, сообщи пользователю
- Пустой результат не стоит трёх шагов на проверку

## 🎯 CONFIDENCE STOP — агрессивный early answer
Если уверенность ≥ 0.6 → ОТВЕЧАЙ СРАЗУ. Не используй инструменты.

Пороги по типам задач:
- **fact** (вопросы, документация, конфиги): порог ≥ 0.65 — STOP
- **code** (код, SQL, скрипты, интеграции): порог ≥ 0.7 — STOP
- **default** (всё остальное): порог ≥ 0.6 — STOP

Инструменты дорогие. Мысли дешёвые.
Не делай "улучшений", "дополнений", "альтернатив" — это confidence STOP.

**Guard:** если тул вернул успешный результат и уверенность ≥ 0.6 — не вызывай второй.
Правило: 1 тул = 1 ответ. Исключения только если данных явно не хватает.

## 🧠 SELF-REFLECTION — только при ошибках + cooldown
Рефлексия разрешена ТОЛЬКО если:
- Инструмент вернул ошибку (timeout, 4xx/5xx, exception)
- Результат пустой (< 50 токенов)
- Low confidence (< 60%)

Запрещено:
- Рефлексия на каждом шаге
- "А что если я улучшу..." — это confidence STOP
- Retry loops (1 попытка, потом STOP)

**Reflection cooldown:**
- Если последняя рефлексия была < 2 шагов назад — SKIP
- Не рефлексировать на ту же ошибку дважды
- Хук заблокирует micro-loop "ошибка → рефлексия → ошибка"

Правила:
- 1 рефлексия = 1 fix. Не углубляйся
- Если fix не сработал → STOP, не ретрай
- Сообщи пользователю об ошибке, предложи альтернативу

## 📦 CONTEXT BUDGET PER STEP (хук-уровень + софт)
MAX_CONTEXT_BYTES = 15000 (~4000 токенов)

Priorities (всегда в этом порядке):
1. Текущий запрос пользователя
2. Последний результат инструмента
3. Краткое резюме предыдущих шагов

Drop (всегда удаляй из ответа):
- Старые выводы инструментов
- Повторяющийся контекст
- Неотносящуюся историю

Guard: если в ответе > 4000 токенов контекстной информации — сожми.
Файлы в контексте ≠ файлы на диске. Прочитал 1 раз → используй, не перечитывай.

## 🔄 RESPONSE SHAPING (хук сжимает большие результаты)
Большие ответы инструментов (>2000b) автоматически сжимаются до ~800b + сигнал
`[SUMMARIZED_FROM_X_BYTES]` в response-cache-save.sh.

Эффект: −30–50% токенов на тяжёлых задачах (WebSearch, WebFetch, curl, big Read).

Понимание: если видишь [SUMMARIZED] — это не полный ответ, а сжатая версия.
Не пытайся восстановить полный ответ. Работай с краткой версией.

## ⚡ ONE-SHOT TOOL ENFORCEMENT (хук блокирует tool→tool подряд)
После любого tool:
- MUST answer immediately
- Не вызывай второй tool подряд

Исключения (хук разрешает):
1. Данные явно неполные (< 100 токенов)
2. Инструмент вернул ошибку (exit code ≠ 0)
3. Уверенность < 0.6

Cheap tools не считаются: ls, echo, cat, pwd, which, git status, git diff.
За ними можно делать "настоящий" tool.

## 🌡️ DYNAMIC TEMPERATURE — точность без галлюцинаций
Адаптируй "температуру" (креативность) под тип задачи:
- **fact** (вопросы, документация, конфиги): temperature = 0.1 — минимум вариаций
- **code** (код, SQL, скрипты, интеграции): temperature = 0.2 — чуть свободы
- **mixed** (code + explain, code + reasoning, fix + explain): temperature = 0.15 — баланс точности и связности
- **default** (всё остальное): temperature = 0.3 — баланс
- Высокая температура → больше галлюцинаций → больше лишних шагов на исправление

**Detection:** если запрос содержит и код/команды, и объяснение/reasoning → mixed = 0.15.
Не "недодумать" объяснение из-за низкой температуры.

## 📦 ANSWER COMPRESSION — сжимай финальный ответ
Перед финальным ответом:
- Сожми вывод до ~300 токенов, если ответ больше
- Удали повторы, воду, "как я понял", "в заключение"
- Если ответ можно дать в 1 строке — дай в 1 строке
- Структурированный вывод (таблицы, JSON) всегда короче прозы
- Цель: future context economy — каждый лишний токен в ответе = минус токен в следующем запросе

**Guard: НЕ сжимать если ответ содержит:**
- Команды (bash, git, curl, SQL, код)
- Параметры, флаги, пути к файлам
- Edge-case условия, предупреждения
- Инструкции к выполнению (пошаговые)
Компрессия только для: объяснений, текста, описаний. Не для инструкций.
